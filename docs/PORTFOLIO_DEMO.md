# Portfolio Demo - Mu Trading Bot

This guide shows a safe local demo flow for presenting Mu Trading Bot as a backend engineering portfolio project.

The demo uses local API requests and demo data. It does not execute trades and it is not financial advice.

## 1. Start API

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
.venv/bin/uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

## 2. Open Swagger

Open:

```text
http://127.0.0.1:8000/docs
```

Swagger should show modules such as System, Dashboard, Watchlist, Scanner, Alerts, Confirmations, Decisions, Backtesting, Scheduler, Maintenance, and Webhooks.

## 3. Import S&P 500

```http
POST http://127.0.0.1:8000/watchlist/import-sp500
```

This imports S&P 500 tickers into the watchlist. The endpoint uses a public source and stores tickers locally.

## 4. Check Dashboard

```http
GET http://127.0.0.1:8000/dashboard/summary
```

Use this to show the current state of watchlist, alerts, decisions, backtesting, and scheduler.

## 5. Run Scanner Batch

```http
POST http://127.0.0.1:8000/scanner/run-watchlist?allow_after_hours=true&debug=true&limit=10&offset=0
```

This scans only one small batch. `allow_after_hours=true` creates watchlist-style opportunities outside market hours, only when the scanner finds a valid setup.

## 6. Check Alerts

```http
GET http://127.0.0.1:8000/alerts/active
```

```http
GET http://127.0.0.1:8000/alerts/watchlist
```

Use these endpoints to show how operational alerts and after-hours watchlist alerts are separated.

## 7. Run Scheduler Once

```http
POST http://127.0.0.1:8000/scheduler/run-once
```

This executes one scheduler cycle. Outside pre-close time, it runs scanner logic or returns market closed, depending on market status.

## 8. Force Pre-Close Demo

```http
POST http://127.0.0.1:8000/scheduler/run-once?force_pre_close=true
```

`force_pre_close=true` is for demo and development validation only. It allows the pre-close confirmation workflow to be demonstrated without waiting for the real market window. It should not be used as a production trading shortcut.

## 9. Check Decisions

```http
GET http://127.0.0.1:8000/decisions
```

This shows the decision history generated by the confirmation workflow.

## 10. Run Backtesting

```http
POST http://127.0.0.1:8000/backtests/run?days=10
```

Backtesting evaluates `COMPRAMOS` decisions using later price data.

## 11. Check Backtesting Summary

```http
GET http://127.0.0.1:8000/backtests/summary
```

This summarizes target hits, stop hits, unresolved outcomes, errors, win rate, and average PnL percent.

## Notes for Portfolio Recording

- Use demo data only.
- Do not show private Discord webhook URLs.
- Do not show `.env` values.
- Do not claim that the bot predicts markets or guarantees profit.
- Present the project as backend architecture, API design, scheduling, data processing, and testing work.
